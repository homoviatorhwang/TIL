## component 통신 방식
- 컴포넌트 : 영역별로 관리되는 화면 코드
- 컴포넌트 사이에 관계 및 규칙이 생김

- 컴포넌트는 각각 고유한 데이터 유효범위를 가짐
- 공유하기 위해서는 props와 이벤트를 이용해야함
상위 컴포넌트에서 하위 컴포넌트로 props 전달, 데이터를 내려줌
하위 컴포넌트에서 상위 컴포넌트로 이벤트 발생, 이벤트를 올려줌

컴포넌트 통신 규칙이 없다면 컴포넌트들 간의 데이터 관계가 생겼을 때 특정 데이터가 바뀌어 오류가 난다면 tracking이 어려움<br>

데이터는 항상 내려오고 아래에서 위로는 이벤트가 올라가므로 tracking이 쉬워짐


 위에서 아래로 내려가는게 **Props** <br>
 아래에서 위로 올라가는게 **Event**

 props는 아래로 흐르기 때문에 윗 부분을 변경할 때 아랫 부분동 변경됌
